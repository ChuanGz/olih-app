# Security Policy

## Scope

This repository is shared for learning and reference. It is not represented as production-ready.

Even so, known secrets, live credentials, signed URLs, and private key material should never remain in the repository.

## Reporting

- Do not post live credentials, exploit chains, or private security details in a public issue.
- Use GitHub private vulnerability reporting if it is available for this repository.
- If private reporting is not available, contact the repository owner through GitHub first and keep the initial report minimal.

## What To Report

- exposed credentials, tokens, or signed URLs
- hardcoded passwords or signing keys
- unsafe configuration defaults that look real
- obvious remote-code, auth, or data exposure risks in the checked-in code

## Response Model

- best effort only
- no SLA
- no guarantee of a patch, support window, or release timeline

## Safe Contribution Rule

If you submit a fix, do not include real secrets in examples, screenshots, tests, logs, or configuration files.
