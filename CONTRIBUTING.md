# Contributing

This repository is maintained primarily as a reference and learning resource. It is not an open collaboration project and it is not a support desk.

## Before You Open Anything

- Read [LICENSE](LICENSE), [DISCLAIMER.md](DISCLAIMER.md), and [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md).
- Use issues for factual bugs, broken instructions, or narrow technical corrections.
- Ask for approval before spending time on a large pull request.

## What Is Welcome

- precise bug reports
- documentation fixes
- narrowly scoped corrections with a clear explanation
- respectful technical discussion that improves the repository

## What Is Not Welcome

- copied code from other sources without clear rights
- low-signal complaints or dismissive commentary
- insulting, humiliating, or baiting comments
- unsolicited large feature work that creates maintenance debt
- attempts to reuse this repository as a drop-in base for another product without written permission

## Contribution Terms

By submitting any issue, pull request, patch, or other material, you confirm that:

- you have the right to submit it
- it does not knowingly infringe another party's rights
- you understand there is no obligation to review, merge, answer, or maintain it

If you submit code, text, diagrams, or other material, you grant the repository owner a perpetual, worldwide, irrevocable, royalty-free, sublicensable right to use, modify, adapt, publish, redistribute, relicense, or reject that submission without compensation.

Do not submit material if you do not agree with these terms.

## Commit Messages

Use short, clean English commit messages with a clear action.

Recommended style:

- `Add configuration guide for local development`
- `Fix broken README links`
- `Sanitize checked-in development settings`
